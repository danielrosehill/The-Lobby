What's going on here

The signature offers two equivalent entry points to the same booking flow:

TrackWho clicksWhat happens
🤖 Agent The recipient's AI agent (or a power user who delegated to one) Resolves joe@joesoap.com via The Lobby, fetches the signed skill manifest, opens an authenticated session, books over MCP / A2A. Both principals get a signed receipt.
👤 Human The recipient themselves Standard scheduling page. Same calendar, no protocol involved.

The two tracks are intentionally peers, not "primary" + "fallback." This signals to the recipient that delegating to an agent is a first-class option, and gives mail clients / agent runtimes a stable visual idiom to recognise: "this sender publishes an agent endpoint."
